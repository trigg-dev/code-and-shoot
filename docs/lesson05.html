LESSON 05

画面の端で止まろう

キャラが画面の外に飛び出さないようにしよう！

今回のゴール 1. 前回の問題点 2. ifの中にif 3. 端の値を考えよう 4. 残りも書いてみよう デモ

今回のゴール

  • ifの中にifを書く「ネスト」をおぼえる
  • 上下左右すべてに端チェックを入れる
  • キャラが画面の外に出ないようにする

1 前回の問題点

前回のレッスンで、矢印キーでキャラを動かせるようになったよね。でも、ずっと同じ方向に動かすと……キャラが画面の外に消えちゃう！

これはカッコ悪い！ゲームっぽくするには、端まで来たら止まるようにしないとね。

2 ifの中にif（ネスト）

「もし左キーが押されていたら動く」だけだと、どこまでも動いちゃう。そこで、もうひとつ条件を追加しよう。

「もし左キーが押されていたら、さらに、もし x が 20 より大きかったら動く」

こうすれば、x が 20 以下になったら動かなくなる＝端で止まる！

if (keyIsDown(LEFT_ARROW)) {
  // さらに条件を追加！
  if (x > 20) {
    x = x - 4;
  }
}

if の中に if を書くことを「ネスト」って言うよ。入れ子みたいなイメージだね。

💡 インデント（字下げ）を使って、どの if がどの {} に対応するか分かりやすくしよう！

3 端の値を考えよう

画面サイズは 横480 × 縦640。キャラのサイズは 横36 × 縦44 の楕円だよ。

ellipse は中心の座標で描くから、端にちょうど止めるには半分のサイズぶんだけ余裕が必要だね。

ざっくり計算すると：

  • 左の端：x > 20（36の半分 ≒ 18、ちょっと余裕をもって20）
  • 右の端：x < 460（480 - 20）
  • 上の端：y > 20（44の半分 ≒ 22、ちょっと余裕をもって20）
  • 下の端：y < 620（640 - 20）
数字を変えてみよう！たとえば 20 を 50 にすると、もっと内側で止まるよ。

4 残りの方向も書いてみよう

左のチェックはステップ2で書いたね。同じパターンで残りの3方向も書いてみよう！

右の端は x < 460 で止まるよ。同じように書いてみよう！
上下も同じパターンで書いてみよう！上の端は y > 20、下の端は y < 620 だよ。
💡 パターンは全部同じ！外側の if でキーを調べて、内側の if で端チェック。方向と比べる数字が違うだけだよ。
スピードの 4 を 8 にして、端ギリギリの動きを確認してみよう！
