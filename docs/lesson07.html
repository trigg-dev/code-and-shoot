LESSON 07

敵を出そう

上から降ってくる敵キャラを登場させよう！

今回のゴール 1. 敵って何が必要？ 2. 敵の変数 3. 敵を動かす 4. リスポーン 5. 敵を描こう 発展チャレンジ デモ

今回のゴール

  • 敵キャラを変数で管理するパターンをおぼえる
  • random() でランダムな位置に敵を出す
  • 画面の外に出た敵をまた上から登場させる
1

敵って何が必要？

前回、弾を作ったときのことを思い出してみよう。弾には4つの変数が必要だったよね：

  • 位置（X座標、Y座標）
  • 速さ（bulletSpeed）
  • 状態（bulletActive）

敵もまったく同じパターン！位置・速さ・状態の変数を作れば管理できるんだ。

💡 プログラミングでは、同じ「パターン」が何度も出てくるよ。弾で覚えたやり方が、そのまま敵にも使えるんだ！
2

敵の変数を準備しよう

弾のときと同じように、敵用の変数を4つ作ろう。プレイヤーや弾の変数の下に書いてね：

let enemyX = 240;       // 敵のX座標
let enemyY = -20;       // 敵のY座標（画面の上の外）
let enemySpeed = 2;     // 敵の速さ
let enemyActive = true; // 敵がいるかどうか

enemyY = -20 にしているのは、最初は画面の上の外にいて、そこから降りてくるようにするため。

弾と敵のくらべっこ

弾：bulletX, bulletY, bulletSpeed, bulletActive
敵：enemyX, enemyY, enemySpeed, enemyActive

名前が違うだけで、構造はまったく同じだね！

3

敵を動かそう

弾は上に飛んだよね（bulletY = bulletY - bulletSpeed）。敵は逆で、上から下に降りてくる。だから Y を増やす：

if (enemyActive) {
  enemyY = enemyY + enemySpeed;
}

これを draw() の中に書こう。弾の処理のあとがいいよ。

💡 弾は - bulletSpeed で上へ、敵は + enemySpeed で下へ。+ と - が逆なだけ！
4

画面の外に出たら、また上から！

弾は画面の外に出たら bulletActive = false で消したよね。でも敵はちょっと違う。敵は何度も出てきてほしい！

だから、画面の下に消えたらY を上に戻して、X をランダムな位置に変えるよ。

if (enemyY > 660) {
  enemyX = random(20, 460);
  enemyY = -20;
}

660 は画面の高さ（640）よりちょっと下。完全に消えてから戻すためだね。

random() ってなに？

random(20, 460) は、20〜460 の間のランダムな数字を返してくれる命令だよ。

サイコロみたいなもの！毎回ちがう場所から敵が出てくるから、ゲームっぽくなるよ。

random(20, 460) の数字を変えてみよう！random(200, 280) にしたら敵はどのあたりに出てくるかな？
5

敵を描こう

弾を描いたときと同じパターンだよ。enemyActive が true のときだけ描こう。

チャレンジ：弾の描画コードを参考にして、敵の描画を書いてみよう！色は fill(255, 80, 80)（赤っぽい色）、サイズは 36, 36（まん丸）がおすすめ！
if (enemyActive) {
  fill(255, 80, 80);
  ellipse(enemyX, enemyY, 36, 36);
}
💡 弾の描画と見くらべてみよう。bulletActive → enemyActive、bulletX → enemyX に変えるだけ！
★

発展チャレンジ：スピードもランダムに！

今のままだと、敵は毎回同じ速さで降りてくるよね。ちょっと退屈かも？

ステップ4のリスポーン処理の中で、速さもランダムに変えてみよう！

リスポーンのとき、enemySpeed も random() で変えてみよう！速さは 1〜4 くらいがちょうどいいよ。
💡 ヒント：enemyY = -20; の下に、enemySpeed = random(1, 4); を追加するだけ！
もっとやりたい人は、敵のサイズもランダムにしてみよう！let enemySize = 36; という変数を追加して、リスポーンのときに変えてみて。
